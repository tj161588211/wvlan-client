## 1. 产品概述和定位

### 1.1 产品概述
WireGuard LAN Manager (WLM) 是一款基于 Windows 桌面的虚拟局域网（Virtual LAN）管理应用。它基于 WireGuard 内核协议，通过简化的配置导入流程，实现用户间“一键组网、自动互通”的虚拟网络体验。

### 1.2 产品定位
- **目标用户**: 需要快速搭建安全、稳定虚拟局域网的个人用户、小型团队、远程协作小组。
- **核心价值**: 降低 WireGuard 的使用门槛，无需手动配置网络参数，通过共享配置文件实现自动组网。
- **技术架构**:
  - **前端**: WPF (.NET/C#) 提供现代化桌面交互界面。
  - **后端**: C++ 集成 WireGuard 驱动，负责网络层操作。
  - **控制层**: 中心化部署的控制服务器，负责节点发现、配置分发、NAT 穿透协调。
  - **网络穿透**: STUN 协议优先，失败时自动切换至 Relay 中继模式。

### 1.3 竞品分析
- **传统 WireGuard 客户端**: 配置复杂，需手动输入 IP、端口、公钥等参数。
- **Tailscale/ZeroTier**: 功能强大但依赖官方服务器，部分高级功能需付费，且配置逻辑对普通用户不够直观。
- **WLM 差异化**: 专注于“单配置文件组网”场景，极简导入流程，完全本地化控制（用户持有配置），支持私有化部署控制服务器。

## 2. 用户角色和使用场景

### 2.1 用户角色
| 角色 | 描述 | 核心诉求 |
| :--- | :--- | :--- |
| **普通用户** | 需要加入虚拟局域网的个人或团队成员 | 快速导入配置，自动连接，无需懂网络技术 |
| **网络管理员** | 负责生成和分发配置文件的团队负责人 | 一键生成配置，管理节点状态，监控网络健康 |
| **系统运维** | 部署和维护控制服务器的技术人员 | 服务器稳定性，日志监控，Relay 流量控制 |

### 2.2 使用场景
#### 场景一：快速组网（核心场景）
1. **背景**: 张三和李四需要安全地共享文件，两人都在不同的家庭网络下。
2. **动作**:
   - 张三在 WLM 中点击“生成配置”，获得一个配置文件（.conf 或二维码）。
   - 张三将文件发送给李四。
   - 李四在 WLM 中点击“导入配置”，选择文件。
   - WLM 自动解析配置，连接控制服务器，完成 NAT 穿透或 Relay 切换。
   - 两人自动获得虚拟 IP，网络互通。
3. **结果**: 30 秒内完成组网，无需手动输入任何 IP 或端口。

#### 场景二：网络状态监控
1. **背景**: 张三需要确认李四是否在线，以及当前连接质量。
2. **动作**: 张三打开 WLM 主界面，查看“节点列表”。
3. **结果**: 界面显示李四的状态（在线/离线）、延迟、连接模式（P2P/Relay）。

#### 场景三：配置更新
1. **背景**: 张三需要添加新成员王五。
2. **动作**: 张三在“配置管理”中点击“添加节点”，生成新配置发送给王五。
3. **结果**: 王五导入后，自动加入现有网络，张三和李四无需重启应用。

## 3. 核心功能清单和优先级

### 3.1 功能优先级定义
- **P0 (Must Have)**: MVP 版本必须实现，否则产品无法运行。
- **P1 (Should Have)**: 重要功能，影响用户体验，MVP 后优先迭代。
- **P2 (Nice to Have)**: 锦上添花功能，后续版本考虑。

### 3.2 功能清单
| 功能模块 | 功能点 | 优先级 | 描述 |
| :--- | :--- | :--- | :--- |
| **配置管理** | 配置文件导入 | P0 | 支持导入 .conf 文件或粘贴配置文本 |
| | 配置文件生成 | P0 | 生成包含当前节点信息的配置文件 |
| | 配置解析与校验 | P0 | 自动解析 WireGuard 标准配置，校验格式 |
| **网络连接** | 自动组网 | P0 | 导入配置后自动连接控制服务器并建立隧道 |
| | NAT 穿透 (STUN) | P0 | 优先尝试 P2P 直连 |
| | Relay 中继 | P0 | STUN 失败时自动切换至中继模式 |
| | 断线重连 | P0 | 网络波动时自动重连 |
| **状态监控** | 节点列表展示 | P0 | 显示所有在线/离线节点及状态 |
| | 连接质量监控 | P1 | 显示延迟、丢包率、传输速率 |
| | 日志查看 | P1 | 查看连接日志和错误信息 |
| **系统设置** | 网络接口管理 | P0 | 自动创建/删除 WireGuard 虚拟网卡 |
| | 开机自启 | P1 | 系统启动时自动连接 |
| | 代理设置 | P2 | 支持 HTTP/SOCKS 代理连接控制服务器 |
| **安全** | 密钥管理 | P0 | 本地安全存储公私钥 |
| | 配置加密 | P1 | 配置文件本地加密存储 |

- **极简主义**: 核心操作一步到位，减少点击次数。
- **状态可视化**: 用颜色（绿/黄/红）和图标直观展示连接状态。
- **技术透明**: 普通用户看不到复杂的网络参数，但提供“高级设置”入口。

## 6. 数据模型定义

### 6.1 配置文件结构 (WireGuard Standard)
```ini
[Interface]
PrivateKey = <本地私钥>
Address = <虚拟IP>/24
DNS = 8.8.8.8

[Peer]
PublicKey = <对端公钥>
AllowedIPs = 0.0.0.0/0
Endpoint = <控制服务器地址>:51820
PersistentKeepalive = 25
```

### 6.2 本地数据库模型 (SQLite)
**表名: `network_profiles`**
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `id` | INTEGER | 主键，自增 |
| `name` | TEXT | 配置文件名称 (默认：导入时间) |
| `config_content` | TEXT | 完整的 WireGuard 配置文本 |
| `private_key` | TEXT | 解密后的私钥 (加密存储) |
| `public_key` | TEXT | 本地公钥 |
| `endpoint` | TEXT | 控制服务器地址 |
| `created_at` | TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMP | 更新时间 |

**表名: `peers`**
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `id` | INTEGER | 主键 |
| `profile_id` | INTEGER | 关联的配置文件 ID |
| `public_key` | TEXT | 对端公钥 (唯一索引) |
| `allowed_ips` | TEXT | 允许的 IP 段 |
| `endpoint` | TEXT | 对端真实 IP:Port (NAT 穿透后) |
| `status` | INTEGER | 0:离线, 1:在线 (P2P), 2:在线 (Relay) |
| `last_handshake` | TIMESTAMP | 最后一次握手时间 |
| `rx_bytes` | BIGINT | 接收字节数 |
| `tx_bytes` | BIGINT | 发送字节数 |

### 6.3 节点状态枚举
```csharp
public enum NodeStatus
{
    Offline = 0,      // 未连接或断开
    Connecting = 1,   // 正在连接中
    P2P = 2,          // P2P 直连成功
    Relay = 3,        // 中继模式
    Error = 4         // 连接错误
}
```

## 7. MVP 版本范围界定

### 7.1 包含功能 (In Scope)
- **平台**: 仅支持 Windows 10/11 (x64)。
- **配置**: 仅支持标准 WireGuard `.conf` 文件导入/导出。
- **网络**: 
  - 基于中心化控制服务器的节点发现。
  - STUN 穿透 + Relay 兜底。
  - 单配置文件组网（即所有节点共享同一套配置逻辑，类似“局域网”模式）。
- **界面**: 
  - 基础 Dashboard（节点列表、状态）。
  - 配置导入/生成向导。
  - 基础日志查看。
- **驱动**: 集成 WireGuard 官方驱动安装包。

### 7.2 不包含功能 (Out of Scope)
- **多网络管理**: 不支持同时管理多个独立的虚拟网络（MVP 仅支持单网络）。
- **跨平台**: 不支持 macOS/Linux。
- **高级路由**: 不支持自定义路由表、防火墙规则配置。
- **图形化网络拓扑**: 仅列表展示，无拓扑图。
- **移动端**: 无 iOS/Android 客户端。
- **用户账户系统**: 无需注册登录，配置即身份。

## 8. 后续迭代方向

### 8.1 短期迭代 (v1.1 - v1.2)
- **多网络支持**: 允许用户创建/加入多个独立的虚拟局域网。
- **跨平台**: 启动 macOS 版本开发（基于 .NET MAUI 或 Avalonia）。
- **网络诊断工具**: 集成 Ping, Traceroute, 端口扫描功能。
- **配置模板**: 提供“文件共享模式”、“游戏联机模式”等预设模板。

### 8.2 中期迭代 (v2.0)
- **P2P 优化**: 引入 ICE 框架，提升 NAT 穿透成功率。
- **流量统计**: 详细的流量图表和月度报告。
- **权限管理**: 管理员可控制节点访问权限（ACL）。
- **插件系统**: 支持第三方插件扩展功能（如聊天、文件传输）。

### 8.3 长期愿景
- **去中心化架构**: 探索 DHT 去中心化节点发现，减少对控制服务器的依赖。
- **企业版**: 支持私有化部署控制服务器，集成 SSO，审计日志。
- **物联网支持**: 优化低功耗设备连接，支持 ARM 架构。
